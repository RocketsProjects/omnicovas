## Summary

Phase 3 is the UI Shell—the moment OmniCOVAS becomes a usable application. Every broadcast from Phase 2 (Pillar 1) is now observable. The application launches, connects to the live telemetry stream, displays ship state in real time, and surfaces critical events both in the main window and in an in-game overlay. The first-run wizard makes setup approachable; the privacy and settings pages put the commander in control; the activity log audits every action the system takes.

## What Shipped (15 Features, Locked)

### **P1 Core (9 features, all required)**

1. ✅ **Tauri Desktop App Shell**
   Mature window management, navigation bar, content routing. Single-window Phase 3; multi-window Tauri support proven (shell + overlay).

2. ✅ **Game Overlay (Transparent Always-On-Top)**
   Secondary Tauri window, click-through by default, hotkey toggle (Ctrl+Shift+O), no input theft from game. Tested with Elite Dangerous fullscreen-borderless.

3. ✅ **FastAPI Bridge (Python ↔ Tauri)**
   Extension of Phase 1 foundation. New endpoints: `/pillar1/ship-state`, `/pillar1/loadout`, `/pillar1/cargo`, `/pillar1/heat`, `/pillar1/pips`. WebSocket `/ws/events` forwards all Phase 2 broadcasts.

4. ✅ **First-Run Onboarding (Three-Path Model)**
   - **Path A (Easy Mode):** 60-second preset selection; Casual default, all privacy OFF, overlay enabled with critical events.
   - **Path B (Custom Setup):** Guided 5-15 minute wizard; preset, AI provider, privacy toggles, output mode, overlay position.
   - **Path C (Power User):** Direct access to Settings + Privacy pages; no defaults applied until commander saves.
   - First-run flag persists in DPAPI-encrypted vault; wizard never re-triggers.

5. ✅ **Privacy Page with Opt-In Toggles**
   Six toggles, all defaulting OFF: EDDN submission (Phase 5+), EDSM tracking (Phase 5+), Squadron sync (Phase 7), AI provider API calls (Tier 3, Phase 4+), Crash reports (structural), Usage analytics (structural). Plain-language descriptions. Export and Delete buttons with GDPR alignment.

6. ✅ **Settings Panel (Three-Tier Customization)**
   - **Tier 1:** Preset profiles (Combat, Explorer, Trader, Miner, Casual)
   - **Tier 2:** Category toggles (Ship Telemetry on; Combat/Exploration/Trading/Squadron/Engineering/Powerplay greyed out with phase captions)
   - **Tier 3:** Granular controls (event toggles, auto-dismiss timers, overlay opacity/anchor/hotkey, AI provider selection, resource budget overrides)

7. ✅ **Activity Log UI (Full)**
   1000-entry ring buffer, filterable by category, searchable, paginated (50 rows/page), exportable to JSON, clearable with confirmation. Audit trail of every Phase 1 broadcast.

8. ✅ **Data Flow Transparency Map**
   `/privacy/data-flows` view. Lists every potential outbound flow: destination, frequency, purpose, current state (active/disabled/not yet shipped). Phase 3 baseline: zero active flows (local-only). Live call counters ready for Phase 5+.

9. ✅ **Accessibility Framework (Principle 1)**
   Semantic HTML across all views. ARIA labels on every interactive element. Focus order logical. Visible focus indicators (`:focus-visible` orange outline). Live regions for status updates. NVDA smoke test passed (documented in `docs/accessibility/nvda_smoke_test.md`).

### **P2 Polish (6 features, delivered)**

10. ✅ **Color-Blind Palette Support**
    Three palettes (deuteranopia, protanopia, tritanopia) as token overrides. Selectable in Settings (Tier 3). Critical banner distinguishable in all modes.

11. ✅ **Font Size Adjustments**
    Three sizes: small (14px), default (16px), large (20px). Controlled via token `--font-size-base`. rem-relative scaling prevents breakage. No layout overflow at large.

12. ✅ **Keyboard-Only Navigation**
    Tab order logical across all views. Enter/Space invoke actions. Esc dismisses modals. No keyboard traps. Shortcut list in Settings.

13. ✅ **Resource Dashboard UI**
    `/resources` route. Live memory, CPU, disk sparklines (last 60 seconds). Budget indicators with warn/critical color thresholds. Reads existing Phase 1 `/resources` endpoint.

14. ✅ **Credits / Contributors Page**
    `/credits` route. Contributors (from git log), dependencies + licenses, required attributions (EDDN, EDSM, Inara, Spansh, EDAstro, BGS-Tally, EDDI, VoiceAttack). Trademark Disclaimer rendered prominently: "OmniCOVAS is not an official tool and is not affiliated with Frontier Developments."

15. ✅ **SBOM Display**
    `/credits/sbom` panel. Software Bill of Materials (CycloneDX format) generated by build pipeline, rendered in UI.

## Architectural Deliverables

### **Phase 3 Patterns & Extensions**

1. **Pattern 1: WebSocket-First State** — UI subscribes to `/ws/events`, renders from broadcast stream. `/state` polling is fallback-only.

2. **Pattern 2: Confirmation Gate UI** — Component renders advisory, "Why?" affordance, explicit Confirm/Decline. Framework landed; Tier 3 content awaits Phase 4+.

3. **Pattern 3: Privacy-by-Default Toggles** — Every outbound flow has explicit toggle on Privacy page. All default OFF. Toggle state persists to DPAPI vault.

4. **Pattern 4: Accessibility from Day One** — Semantic HTML, ARIA labels, focus order, visible indicators baked in from first commit.

5. **Pattern 5: Three Output Modes Router** — Voice / Overlay / Both. Phase 3 owns Overlay path; Voice stubbed (greyed out, Phase 3+ caption).

6. **Pattern 6: Three-Path Onboarding** — Single wizard, three branches: Easy (60s), Custom (5-15m), Power User (commander-paced). First-run flag prevents re-trigger.

## Known Limitations

1. **Overlay multi-monitor:** Single-monitor only. v1.1 will add per-monitor positioning.

2. **Canvas chart accessibility:** Heat sparkline is canvas; mitigation uses `aria-label` + separate text value. v1.1 may switch to SVG for native accessibility.

3. **CAPI placeholder:** Frontier approval pending. UI shows "Coming when approved" messaging.

4. **Voice greyed out:** EDDI integration deferred. Phase 3+ will activate after VoiceAttack/EDDI pairing.

5. **Local LLM latency:** Qwen inference on RTX 4070 Laptop may exceed latency budgets if model is busy. Phase 3 defaults to NullProvider. Gating added in Settings.
