- Reasons list and confidence.
- No hard-coded ED mechanics unless KB-verified.

- omnicovas/features/combat_threat.py or combat/threat.py

- tests/test_combat_threat.py
- omnicovas/api/pillar2.py

[ ] Define inputs: hull/shields/modules/fuel/heat/pips from Pillar 1, target snapshot, active interdiction, hostility warning, CZ state, munitions known/unknown.
[ ] Define output: threat_level, score or enum, reasons, confidence, inferred, timestamp.
[ ] Use KB entries for thresholds. If a threshold is not already verified, add a KB entry only with patch/source/confidence/_justification or withhold the rule.
[ ] Explicitly label inferred threat. Do not overwrite raw telemetry state.
[ ] Publish COMBAT_THREAT_CHANGED only on level change or reason change.
[ ] Add /pillar2/threat endpoint.
[ ] Add tests for null-heavy, low, medium, high inputs using internal state objects, not fake ED raw events.
