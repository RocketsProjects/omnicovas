## 🎮 Section 2 — Frontier Developments

### 2.1 Elite Dangerous EULA

**Prohibited under Law 9 Interpretation B:**
- ❌ Modifying game executable
- ❌ Injecting into game process
- ❌ Auto-aim, auto-pilot, auto-trade
- ❌ Bot play or AFK farming
- ❌ Account sharing facilitation
- ❌ Anti-cheat circumvention
- ❌ Redistributing game assets

**Permitted:**
- ✅ Reading journal files (documented for tools)
- ✅ Reading Status.json (documented live state)
- ✅ Reading ModulesInfo.json, Cargo.json, Backpack.json, ShipLocker.json (documented)
- ✅ Using CAPI per Frontier's terms (with approval)

**Compliance Status:**
- ✅ All "actions" are advisory (Law 1 Confirmation Gate)
- ✅ No automation touches game input
- ✅ Each user has own Frontier account
- ✅ Game files read-only

### 2.2 Frontier CAPI Terms

**Requirements:**
- ⏳ Developer approval required (Phase 3 submission)
- ✅ OAuth2 PKCE authentication
- ✅ Rate limits: ~1 query/min guidance, hard stop 2/sec
- ✅ 25-day refresh token expiration
- ✅ No abuse or scraping

**Architecture Compliance:**
- ✅ OAuth2 PKCE designed
- ✅ Rate limits in Registry
- ✅ FC polling aligned with 15-min server refresh
- ✅ Token refresh automated
- ✅ Graceful 422 handling

### 2.3 Trademarks

**Must Not:**
- Imply official partnership
- Use trademarks in ways that confuse users
- Use Frontier logos without permission

**Will Do:**
- ✅ Display required disclaimer
- ✅ Use only the words "Elite Dangerous" for game reference, no logos
- ✅ Use "OmniCOVAS" as our own trademark

## 🚨 Section 8 — Compliance Red Flags

### Critical Violations (Immediate Stop)

- ❌ Exceeding any API's rate limit
- ❌ Injecting into Elite Dangerous game process
- ❌ Auto-pilot or auto-combat
- ❌ Bundling EDDI or other GPL-3.0 code
- ❌ Logging API keys or OAuth tokens
- ❌ Sharing CMDR data without explicit opt-in
- ❌ Data outbound without Privacy-by-Default opt-in

### Warning-Level Issues

- 🟡 Cache TTL too short
- 🟡 User data flow without clear disclosure
- 🟡 Dependencies with incompatible licenses
- 🟡 Missing attribution
