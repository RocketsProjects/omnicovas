# PB05-05 — Ship Schematic Registry and Sidewinder SVG MVP

OBJECTIVE
Create the internal ship schematic system for Dashboard use:
- original top-down Sidewinder technical wireframe
- generic fallback schematic
- ship key mapping/registry
- accessible hotspot metadata
- safe SVG rendering through DOM APIs

This playbook does NOT replace the Dashboard layout yet. It builds the schematic component and tests it.

ALLOWED FILES
Create:
- ui/data/ship-schematics.js
- ui/components/ship-schematic.js
- ui/styles/ship-schematic.css
- ui/__tests__/ship-schematic.test.js

Modify:
- ui/index.html, only to load new CSS/JS files
- ui/styles/command-deck.css or components.css, only if needed for host classes

FORBIDDEN
No backend/Tauri/Python/package changes. No copied Frontier assets, screenshots, traced silhouettes, extracted models. No external SVG imports or remote assets. No new dependency. No visible dashboard replacement yet. No fake telemetry.

SCHEMATIC ART REQUIREMENTS
- Top-down blueprint.
- Sidewinder: recognizable but simplified original approximation.
- Generic fallback: abstract technical hull diagram.
- SVG line art authored as original vector geometry in code/data.
- Simple path/polyline/polygon/circle/line primitives.
- No heavy filters or animation loops.
- currentColor/token-driven classes.
- No exact Frontier silhouette tracing.

REGISTRY REQUIREMENTS
window.OmniShipSchematics or equivalent classic-script global:
- resolveShipKey(rawShipType)
- getSchematic(shipType)
- hasSchematic(shipType)
- listSupported()
- sidewinder entry
- generic entry

Each schematic entry:
- id
- displayName
- aliases
- viewBox
- paths/shapes
- hotspots
- defaultPanelAnchors
- supportedSubsystems

Required hotspot IDs:
hull, shield, fuel, heat, cargo, modules, pips.
Optional: ship, rebuy.

HOTSPOT REQUIREMENTS
Hotspots are metadata now, but renderer must be able to render them as accessible controls:
aria-label, panelId, defaultExpanded true, x/y position, visual class/severity-compatible class.

COMPONENT REQUIREMENTS
window.OmniShipSchematic:
- create(shipType, options)
- mount(container, shipType, options)
- resolve(shipType)
- createHotspotButton(hotspot, options)

Use document.createElementNS for SVG and document.createElement for buttons/wrappers.
No innerHTML, outerHTML, insertAdjacentHTML, DOMParser, template.innerHTML.

TESTS
Minimum:
- registry exists
- sidewinder resolves
- unsupported ship resolves to generic fallback
- aliases map to sidewinder where configured
- create returns SVG/wrapper structure
- hotspot buttons are real button elements or accessible controls
- hotspot controls have aria-label, aria-controls/panel linkage, aria-expanded
- no unsafe HTML sinks in ship-schematic.js and ship-schematics.js
- package files unchanged

VERIFICATION
npm test
git grep -n "innerHTML\|outerHTML\|insertAdjacentHTML" -- ui
git grep -n "8000\|OMNICOVAS_PORT || 8000\|window.PORT || 8000" -- ui
git diff -- package.json package-lock.json
git status
