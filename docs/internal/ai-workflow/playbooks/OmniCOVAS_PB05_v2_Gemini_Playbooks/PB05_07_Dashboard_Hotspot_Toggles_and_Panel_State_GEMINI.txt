# PB05-07 — Dashboard Hotspot Toggles and Panel State

OBJECTIVE
Add the first interactive layer to the Dashboard schematic:
- clicking/keyboard-activating ship subsystem hotspots toggles matching callout panels
- panels visible by default
- hidden/shown state resets every launch
- no persistence, no dragging, no snap zones, no floating windows

ALLOWED FILES
Modify:
- ui/views/dashboard.js
- ui/components/ship-schematic.js
- ui/styles/dashboard.css
- ui/styles/ship-schematic.css
- ui/__tests__/dashboard-schematic.test.js
- ui/__tests__/ship-schematic.test.js

Optional:
- ui/index.html only if panel IDs/classes need a narrow static attribute fix

FORBIDDEN
No backend/Tauri/Python/package changes. No localStorage/sessionStorage/settings persistence. No free drag. No snap zones in PB05. No compact mode implementation. No combat logic.

BEHAVIOR REQUIREMENTS
1. All callout panels visible by default on Dashboard load.
2. Hotspot activation toggles visibility of associated panel.
3. Hotspot updates aria-expanded and active visual state.
4. Panel stays in DOM when hidden if possible.
5. Add "Show all systems" / "Reset panels" control.
6. State is in-memory only and resets on reload.
7. If JS cannot initialize toggles, panels remain visible by default.

ACCESSIBILITY
Hotspots are real buttons or keyboard-operable controls with aria-controls, aria-expanded, aria-label. Hidden panels must not confuse screen readers.

TESTS

- fuel hotspot toggles fuel panel and aria-expanded
- second activation shows it again
- reset/show-all restores all panels
- missing target fails safely
- no persistence APIs introduced
