3. Update existing StateManager/session state for each confirmed transition.
4. Publish INTERDICTION_STARTED, INTERDICTION_ESCAPED, INTERDICTION_SUBMITTED, and INTERDICTION_FAILED when grounded by telemetry.
5. Add Activity Log entries for each transition with source event, timestamp, known aggressor/null, and outcome/null.
6. Create escape_planner.py with EscapeOption fields: label, description, confidence, source_refs, requires_user_action.
7. Implement advisory-only option generation. It may return an empty list. It must not include callbacks, commands, keypresses, macros, or executable actions.
