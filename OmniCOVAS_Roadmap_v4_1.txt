## 🗓️ Release Strategy

### v1.0 — "Core Intelligence"
**Goal:** Useful daily tool for commanders. Establishes architecture and trust.

**Scope:**
- Pillar 1 (Ship Telemetry) — complete
- Pillar 2 (Combat) — core features
- Pillar 3 (Exploration) — core features
- Pillar 5 (Trading/Mining) — core features
- Pillar 7 (Squadron) — basic P2P sync only
- UI shell with overlay
- Voice I/O (via EDDI/VoiceAttack) for basic queries
- Settings + privacy page
- First-run onboarding (Three-Path model)

### v1.1 — "Expansion"
Pillar 4 (BGS/PP2) core, Pillar 6 (Engineering) core, feature additions across pillars.

### v1.2 — "Deep Features"
Illegal Ops suite, squadron stretch, Build Template Library.

### v1.5 — "Polish"
Quality, performance optimization, remaining features.

### v2.0 — "Vision Complete"
All 166+ features. Native voice option. Advanced integrations.

# PART XIII — DEPENDENCIES & BLOCKERS (v4.1)

## Resolved Phase 1 Dependencies

| Dependency | Resolution |
|-----------|-----------|
| Python 3.11 install + uv + venv | ✅ Resolved |
| Tauri v2 + Rust toolchain | ✅ Resolved |
| GitHub repo + CI | ✅ Resolved |
| DPAPI access for vault | ✅ Resolved (Windows-only for v1.0) |

## Active Phase 2 Dependencies

| Dependency | Risk | Mitigation |
|-----------|------|-----------|
| Clean journal schema from current ED patch | Low | Defensive Update framework catches schema changes |
| Status.json stability | Low | Already exercised in Phase 1; well-documented |
| KB verification (no external API needed for Pillar 1 mechanics) | Medium | Most Pillar 1 mechanics verifiable via in-game testing |

## Critical Dependencies for Later Phases

| Dependency | Blocks | Mitigation |
|-----------|--------|-----------|
| Frontier CAPI approval | CAPI-dependent features in all pillars | Features gated behind approval, rest ship regardless |
| Inara whitelisting | CMDR lookups, faction data | Ship without, update when approved |
| EDDI (user install) | Voice features | Documentation clear; core features work without |
| VoiceAttack (user purchase) | Voice commands | Optional feature; documentation clear |
| Spansh data dump availability | Route plotting, system intel | If unavailable, fall back to live API (slower) |

## Known Risks

1. **Frontier CAPI approval denied or delayed** — ship v1.0 with Option C (features gated)
2. **Inara rate limit changes** — our architecture already tight against 2/min; resilient
3. **Game update breaks journal schema** — Defensive Update framework catches this
4. **Developer time pressure** — scope flexibility built in; features can defer
5. **AI-accelerated pace does not hold for feature work** — pace may slow when KB verification dominates over code generation. Phase 2 timing may regress toward nominal.
