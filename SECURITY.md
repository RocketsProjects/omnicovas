# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OmniCOVAS, please report it responsibly:

1. **Do NOT open a public GitHub issue**
2. Go to your repository's **Security** tab → **Private vulnerability reporting**
3. Report the vulnerability there

We will:
- Acknowledge receipt within 72 hours
- Investigate and develop a fix
- Release a patched version
- Credit you in the release notes (Hall of Fame)

## Security Commitment

OmniCOVAS follows these security principles:
- API keys are never logged or transmitted
- All external data flows are logged and auditable
- Rate limits are enforced before API calls
- Data stays local by default (opt-in to send anywhere)

Thank you for helping keep OmniCOVAS secure.
